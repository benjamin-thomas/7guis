Technical decisions

7GUIs Timer in Gleam/Lustre, with effects as data and a local time-travel debugger.

Goal

This port keeps the design that made the PureScript/Flame timer useful to study: the update function is pure, side effects are returned as inspectable values, tests assert on those values, and the runtime interpreter is the only impure edge.

Shape

Msg update Model + Instr interpret Effect Msg 
pub fn update(model: Model, msg: Msg) -> #(Model, List(Instr))

`Instr` is the app-specific effect ADT. It describes work such as starting the browser interval, notifying the fake server, or logging. Nothing runs while `timer.update` executes.

Effects as data

Pure core

`src/timer.gleam` owns `Model`, `Msg`, `Instr`, `init`, `update`, and `view`.

Runtime edge

`src/lustre_timer.gleam` maps each `Instr` into a Lustre `Effect`.

Tests

`test/lustre_timer_test.gleam` compares exact `(Model, List(Instr))` values.

StartBtnClicked
-> Model(timer_state: Running(0.0), ...)
-> [StartTimer, ConsoleLog("Timer started")]

Why not call effects directly?

Calling `setInterval`, server code, or logging from `update` would make the state machine hard to test and harder to rewind. Returning instructions keeps the behavior visible and lets the debugger record clean state transitions.

Lustre integration

Lustre already has managed `Effect(msg)` values. This example adds one level above that: domain effects are first-class application data, then the interpreter lowers them into Lustre effects. 

fn update(model, msg) {
  let #(next_model, instructions) = timer.update(model, msg)
  #(next_model, interpret_all(instructions))
}

Timer source

The browser interval lives in `timer_ffi.js`. `StartTimer` creates a recurring source that dispatches `Ticked(100.0)`. `StopTimer` clears it. `NotifyServer` simulates a round trip and dispatches a success message.

Debugger

The app mounts through `lustre_time_travel.application(init, update, view)`. The package wraps the Lustre app, records every transition after the app update, and can replace the current model when a history row is selected. Selecting a row pauses live updates; clicking the overlay resumes.

This debugger is vendored as an internal Gleam package here so the example can be deployed and studied without depending on another local repository. It is intentionally small enough to become a separate package later.

Trade-offs

  • Good: tests assert exact side-effect intent without mocks.
  • Good: the debugger is app-agnostic and works without changing the timer domain module.
  • Cost: `Instr` needs an interpreter, so simple apps have one extra layer.
  • Cost: the debugger uses JavaScript reflection and should be treated as development tooling.
